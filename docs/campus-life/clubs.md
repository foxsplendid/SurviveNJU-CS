# 社团与技术组织

技术社区和集训队可以提供交流与实践机会，但它们的组织性质并不相同。本页只记录有公开来源可追踪的状态，不把 GitHub 组织、群聊或社区自称自动视为学校注册社团，也不根据往届信息推断当前招新方式。

!!! warning "先确认组织性质与当前状态"
    学校或院系正式组织应能在南京大学官方域名下找到对应发布；学生社区可以有稳定的项目和公开账号，但这不等同于校方认证。参与前应从当前公开页面确认维护者、活动规则和联系方式，不使用无法追溯来源的群号或二维码。

---

## 🐧 1. 南京大学 Linux 用户组（NJU-LUG）

**性质：学生技术社区。** 南京大学 e-Science 中心的[官方介绍](https://sci.nju.edu.cn/8e/3a/c30384a560698/page.htm)提到 LUG@NJU 同学参与镜像站升级；[NJU-LUG GitHub 组织](https://github.com/nju-lug)自称 “Nanjing University Linux Users Group”，并持续维护与南京大学相关的开源项目。现有公开来源不足以证明它当前属于学校注册学生社团，因此本页不使用“官方社团”称谓。

可由公开来源确认的项目包括：

*   **开源项目**：[NJUThesis](https://github.com/nju-lug/NJUThesis)、`modern-nju-thesis`、镜像站前端等项目列在该 GitHub 组织下。它们是社区维护项目，不能替代学校或院系当届发布的论文规范。
*   **镜像站协作**：南京大学 e-Science 中心的[官方介绍](https://sci.nju.edu.cn/8e/3a/c30384a560698/page.htm)说明，南京大学开源镜像站由 e-Science 中心运行维护，LUG@NJU 同学曾协助升级站点。当前服务入口为[南京大学开源镜像站](https://mirror.nju.edu.cn/)。因此，镜像站属于学校中心运行的服务，不能简单写成由 NJU-LUG 独立维护。

本轮核验没有找到可公开确认的当前招新通知。希望参与维护或交流时，可从 GitHub 组织及具体仓库当前公开的 Issue、Discussion 或联系方式进入；不要沿用往届群号、公众号名称或“核心技术部”等未经当前来源确认的信息。

## 🧮 2. 南京大学 ACM-ICPC 集训队

**性质：校内程序设计竞赛集训组织，不按普通学生社团介绍。** 南京大学软件研究所[马骏老师个人页](https://ics.nju.edu.cn/people/junma/)列有“南京大学 ACM-ICPC 集训队教练”身份及队伍参赛信息，可以直接证明校内集训队的存在；该来源不能证明固定选拔时间、加入门槛或训练计划。

有意参加时，建议依次核对：

1. 计算机学院或相关院系当学期通知；
2. 教务系统中的当前课程信息；
3. 教练或课程团队当年发布的报名、选拔与训练说明。

不要根据往届经验默认“大一固定选拔”、以 OJ 刷题量直接申请或暑期必有统一集训。

## 🔎 3. 选择与核验建议

*   **学校或院系组织**：优先查找 `nju.edu.cn` 域名下的学院、教师、教务或团学部门页面，并确认发布日期与适用学期。
*   **学生社区**：查看维护者、项目历史、许可证和当前联系方式；公开活跃不代表学校背书，也不保证长期持续。
*   **第三方平台与资源**：GitHub、社交平台和群聊只是承载渠道。不要向非学校系统提交统一身份认证密码、验证码、Cookie 或长期有效 Token。
*   **实际收益**：社团经历、项目协作和竞赛训练的价值因人而异，不应承诺内推、免笔试、升学加分或固定人脉收益。
