# 实习与求职指南

求职目标、岗位要求和招聘批次因行业、公司、年份及岗位而异。学校背景、刷题数量或某一次实习都不能保证进入面试或获得录用；更稳妥的准备方式是对照目标岗位的官方招聘说明补齐能力和材料。

南京大学校内招聘与就业手续可从 [南京大学学生就业指导中心](https://job.nju.edu.cn/) 进入。具体岗位仍应跳转到用人单位当前官方招聘页面核验，不能仅凭转发群、聚合网站或个人“内推”表单提交敏感信息。

---

## 🚀 1. 如何看待暑期实习与转正

暑期实习是了解岗位、积累项目证据和争取后续录用的渠道之一，但并非所有公司、岗位或年份都设置转正环节，也不存在“实习后必然转正”的通用规则。

投递前至少核对：

* 招聘信息是否来自公司官方招聘站、官方公众号或学校就业系统；
* 岗位属于日常实习、暑期实习还是正式校园招聘；
* 工作地点、到岗要求、实习期限和转正安排是否写入正式通知；
* 简历、笔试、面试和录用分别由谁通知，是否能够在官方渠道复核。

!!! warning "招聘节奏不是统一校历"
    不同公司的招聘窗口和名额分配会调整。社区整理的“春招、暑期、秋招时间表”只能用来设置提醒，不能替代雇主当期公告。

## 🛠️ 2. 技术岗位的常见准备面

### ① 算法与编程

部分技术岗位会安排在线编程、现场编码或代码讨论，但轮次、时长、难度和淘汰方式没有统一标准。刷题的目标应是掌握方法并能解释复杂度、边界条件和测试方式，而不是追求一个保证通过的题量。

可按主题练习数组与字符串、链表与树、图搜索、排序与查找、动态规划以及基础数据结构，并通过限时模拟检查代码正确性。

### ② 计算机基础与岗位知识

根据职位描述选择复习范围，常见方向包括：

* **计算机网络**：TCP 状态与可靠传输、HTTP/HTTPS、DNS 和常见网络故障定位；
* **操作系统**：进程与线程、同步、虚拟内存、文件和 I/O；
* **数据库**：索引、事务、隔离级别、查询分析与数据建模；
* **语言与工程**：目标语言的运行时、内存与并发模型，以及测试、调试、版本控制和部署基础。

面试问题通常会围绕简历继续追问。只写自己实际完成、能够复现并解释取舍的项目内容。

## 🗓️ 准备顺序参考

下表按本科阶段给出准备顺序，不承诺任何公司会在对应学期开启招聘。

| 阶段 | 建议动作 | 核验结果 |
| :--- | :--- | :--- |
| **基础阶段** | 完成课程项目，练习数据结构与调试，保留可运行代码和说明文档。 | 项目可以从干净环境运行，能够解释自己的贡献。 |
| **投递前** | 阅读目标岗位说明，按真实经历整理一页简历，准备项目和基础知识问答。 | 简历中的每项事实都有代码、文档或经历支撑。 |
| **招聘开放后** | 只从可验证渠道投递，记录岗位编号、状态和通知来源。 | 能识别虚假内推、付费保录和非官方收集个人信息的页面。 |
| **实习或录用阶段** | 书面确认岗位、地点、期限、薪酬、知识产权和后续考核安排。 | 关键条件以正式文件为准，不依赖口头承诺。 |

!!! tip "用 STAR 组织项目描述"
    可以按 Situation、Task、Action、Result 说明项目。只有在测量方法、基线和个人贡献都能解释时才写量化结果，不要为增强简历效果编造数字。
