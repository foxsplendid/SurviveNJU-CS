# 推免指南

推荐免试包含两个相互独立的环节：本科院校遴选并公示推荐资格，招生单位按本单位办法考核并接收。取得推荐资格不等于已被目标院校录取，参加夏令营或预推免也不替代教育部推免服务系统中的正式流程。

!!! danger "每一届都要重新核对"
    推荐名额、课程计算范围、排名口径、成果认定、申请材料和接收规则都可能调整。任何往年经验都不能代替本年度学校办法、院系细则和目标招生单位通知。

---

## 📐 1. 先分清推荐资格与接收资格

### 本科院校推荐

南京大学发布校级工作办法后，各院系再公布实施细则。学生需要核对：

* 基本资格、排名范围和名额分配；
* 哪些课程计入、成绩如何加权；
* 补考、重修、缓考、交流课程等如何认定；
* 科研、竞赛或其他成果是否计分，以及成果目录、作者顺序、上限和证明材料；
* 公示、复核和异议处理方式。

不存在适用于所有泛 CS 院系的“统一三项权重”。普通成绩单 GPA 也不能直接替代院系推免排名。

!!! example "一份有明确年份的历史样本"
    人工智能学院 [2025 年推荐免试工作实施细则](https://ai.nju.edu.cn/00/f9/c17810a786681/pagem.htm) 面向 2026 届毕业生，明确列出了学业排名基本条件、课程范围、外语条件以及论文和竞赛加分规则。这些条款只说明该院该届如何计算，不能外推到其他院系或后续年份。

### 招生单位接收

获得本校推荐资格后，还要按目标招生单位通知提交申请并参加其规定的考核。夏令营、预推免、面试或导师沟通的名称和效力各校不同；只有按当年接收办法完成申请，并在国家系统中完成规定步骤，才进入正式接收流程。

招生单位可能使用“优秀营员”“拟录取”等表述，但这些名称不是可以跨单位套用的统一状态体系。教育部推免服务系统当前公开说明使用的是“复试通知”和“待录取通知”，最终名单、录取名单和学籍注册以系统备案信息为准。

引用任何状态时，应同时注明招生单位、招生年度、通知标题和官方链接，并保留原通知用词，不把“优秀营员”“拟录取”“待录取”自行换算成前后相接的等级。收到系统通知后，应在当年度系统和招生单位规定的期限内处理；逾期、确认或放弃的具体后果，以当年度书面说明为准。

## 🗓️ 2. 流程清单，而非固定时间表

| 阶段 | 应做的事 | 不能据此推断的事 |
| :--- | :--- | :--- |
| **政策准备** | 关注本校和院系通知，准备成绩、排名、成果及诚信材料。 | 往年排名线不等于本届入选线。 |
| **接收申请** | 逐校阅读夏令营、预推免或接收办法，按官方系统提交。 | 入营、通过考核或导师口头回复不当然等于正式录取。 |
| **国家系统** | 按教育部当年安排查询资格、填报志愿并完成招生单位要求的确认。 | 社区日历和聊天记录不能替代系统状态。 |
| **结果确认** | 保存系统记录和招生单位正式通知，核对培养单位、专业、研究方向及其他条件。 | 未写入正式通知的承诺不应作为放弃其他机会的唯一依据。 |

官方入口：

* [教育部推免服务系统](https://yzst.chsi.com.cn/tm/)
* [中国研究生招生信息网推免流程说明](https://yz.chsi.com.cn/yzzt/tmzn/)

## 🔗 3. 社区资源的正确用法

以下项目可用于发现信息线索或阅读个人经历，但不是招生文件：

* [CS-BAOYAN](https://github.com/CS-BAOYAN)：社区维护的院校信息与截止日期索引；每条信息都应回到招生单位官网复核。
* [NJU-SE-15-share-review/postgraduate-recommendation](https://github.com/NJU-SE-15-share-review/postgraduate-recommendation)：历史个人经验，只能帮助理解准备过程，不能代表当前政策或录取标准。

不要把来源不明的考核题、名单截图、群聊消息或个人预测当作官方信息，也不要传播无法确认授权和保密状态的材料。

## 💡 4. 决策建议

1. **为每个目标项目建立来源表**：记录招生单位、招生年度、通知标题、发布日期、官方链接和最后核验时间。
2. **按通知准备考核**：只有当官方办法明确设置机试、笔试或指定材料时，才据此安排准备；不要预设固定题数或淘汰规则。
3. **正确理解导师沟通**：联系导师可以了解方向、名额和培养方式，但个人回复不能替代院系考核、国家系统和正式录取通知。
4. **保留备选路径**：在正式结果确认前，不要仅凭非正式表述取消其他升学或就业安排。
