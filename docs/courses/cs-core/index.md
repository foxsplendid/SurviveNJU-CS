# 计算机学院相关核心课程

!!! info "本页是知识地图，不是培养方案"
    课程名称、学分、开课学期、必修/选修性质和正式先修关系会随专业、年级和教学班变化。请以本人适用的培养方案、教务系统和课程主页为准；本目录只说明知识之间的常见联系。

## 知识主线

```text
程序设计 -> 数据结构 -> 算法设计与分析

程序设计 + 机器级表示 -> ICS -> 操作系统
数字逻辑 ----------------> 计算机组成与体系结构

程序设计 + 数据结构 + 形式语言 + 系统基础 -> 编译原理

程序设计与系统基础 -> 计算机网络 / 数据库 / 软件工程
```

箭头表示学习上的知识支撑，不代表教务系统中的强制先修。尤其是编译原理不应被写成“完成 OS 后的唯一下一门课”：计算机学院与软件学院还有不同的编译教学路线，应先确认院系和年份。

## 课程页面

### 系统与基础设施

- [计算机系统基础（ICS）](ics.md)：机器级程序、存储系统与 Project-N/PA；实验分支按年份核对。
- [操作系统：设计与实现](os.md)：并发、虚拟化和持久化；2026 MiniLab 主线与 2022 历史 OSLab 已分开说明。
- [计算机组成原理](computer-organization.md)：数据通路、流水线、存储层次与 I/O 的硬件视角。
- [编译原理](compiler.md)：分别说明计算机学院 C--/MIPS 历史实例与软件学院公开课程资料，不混用教师和工具链。
- [计算机网络](networks.md)：分层协议、可靠传输、路由和网络测量。
- [数据库系统](database.md)：关系模型、SQL、事务、索引和查询处理。

### 算法与软件

- [数据结构与算法](data-structures.md)：数据组织、经典算法及复杂度分析。
- [软件工程](software-eng.md)：需求、设计、协作、测试和软件演化的基础方法。

## 使用建议

1. 选课前先核对本人年级培养方案，再用课程主页确认教师、材料、实验和考核。
2. 系统类实验尽早阅读规范并建立最小测试；不要依据第三方经验帖推断工作量、截止时间或评分比例。
3. ICS、OS 和编译作业的协作与代码复用边界不同，提交前逐门阅读学术诚信规则。
4. 遇到版本冲突时，以当年课程组发布内容为准，并在本指南中保留年份标签。
