# 计算机组成原理

*   **课程学分**：3 学分
*   **授课学期**：大二秋季学期
*   **授课教师**：计科系教师组
*   **修读阶段**：计算机系必修课

---

## 🌟 课程简介与地位

如果说《计算机系统基础 (ICS)》是从系统软件和汇编的视角来解剖计算机，那么《计算机组成原理》则是**纯粹从硬件逻辑与数字电路的视角**来探究计算机硬件的设计原理。

本课程聚焦于单周期与多周期 CPU 的数据通路设计、控制单元（CU）的微程序设计、指令级流水线（Pipeline）与冲突处理、存储器层次结构（Cache）以及 I/O 系统与中断机制。它是系统主线中承接数字逻辑电路与后置操作系统的重要基石。

---

## 📚 核心内容与期末考点

### 1. 核心理论考点
*   **数据的表示与计算**：补码加减运算、溢出判定、浮点数表示（IEEE 754 标准，必考）。
*   **CPU 数据通路与控制**：
    *   单周期/多周期 CPU 的数据通路图绘制与控制信号分析。
    *   硬连线控制器与微程序控制器的工作原理。
*   **指令流水线 (Pipeline)**（重难点）：
    *   流水线的段级划分（IF, ID, EX, MEM, WB）。
    *   **流水线冲突（Hazards）分析**（必考大题）：结构冲突、数据冲突（Read-After-Write）、控制冲突。掌握**旁路技术（Bypassing/Forwarding）**和插入气泡（Stall）的解决机制。
*   **存储体系与 Cache**（必考大题）：
    *   Cache 的映射方式：直接映射、全相联映射、组相联映射（Set-Associative）。
    *   地址结构划分（Tag, Index, Offset）。
    *   Cache 替换算法（LRU, FIFO）与命中率计算。
*   **总线与 I/O**：总线控制与仲裁、程序查询方式、中断处理流程、DMA 传输机制。

### 2. 实验要求
本课程的实验通常要求使用硬件描述语言（如 Verilog HDL）在 FPGA 开发板上，或在 Logisim 等逻辑模拟器上，亲手搭建一个单周期和流水线 CPU。你需要让这个 CPU 能够执行若干条基本的 MIPS 或 RISC-V 汇编指令。

## 💡 备考与学习攻略

1.  **攻克 IEEE 754 与 Cache 映射计算题**：
    *   IEEE 754 单精度浮点数的十六进制转换是经典小题；Cache 组相联映射下，地址位数的计算、以及给你一串内存访问序列判定 Cache 是否命中的题目，是期末必考的大分题。
    *   熟记公式：$\text{Cache组数} = \frac{\text{Cache行数}}{\text{路数}}$，并清晰分辨物理地址中各字段的边界。
2.  **手推流水线时序图**：
    *   面对数据冲突题目（如：前一条指令修改了 `$t0`，紧随其后的指令要读取 `$t0`），在草稿纸上画出时序图（各段在不同时钟周期的执行状态），判断是否需要硬件旁路，还是必须插入 Stall。
3.  **连接 ICS 与考研 408**：
    *   本课程的内容与考研 408 中《计算机组成原理》的部分重合度高达 95%。对于准备国内考研的同学，这门课的学习效果将直接决定你 408 专业课的起点。
