*   **课程学分**：4 学分

*   **授课教师**：袁春风、蒋炎岩、汪亮等课程组教师
*   **修读阶段**：计算机系必修 / 泛CS转专业准入课

---

## 🌟 课程简介与地位

《计算机系统基础》（Introduction to Computer Systems, 简称 ICS）是南京大学计算机系乃至全校泛CS专业**最核心、最硬核的王牌奠基课之一**。本课程旨在回答一个最根本的问题：

> **“高级语言编写的程序，是如何在一台由晶体管组成的计算机系统上跑起来的？”**

本课程的理论部分涵盖了数据的二进制表示、程序的机器级表示（x86/RISC-V 汇编）、CPU 译码与执行流、存储层次结构（Cache）、虚拟存储空间管理以及简单链接器。

而本课程的精髓，在于由南大系统组自主开发的 **Project-N (PA) 实验**。

---

## 🛠️ Project-N: 模拟器 NEMU 的开发之路 (PA)

PA (Programming Assignment) 要求你亲手用 C 语言编写一个名为 **NEMU (NJU Emulator)** 的全功能 CPU 模拟器。你需要让它能够模拟 x86 或 RISC-V 指令集，并最终在你自己写的模拟器上运行起来《仙剑奇侠传》和文字游戏。

### 🗺️ PA 实验阶段路线图

```
  PA0: 搭建 Linux 开发环境 & 熟悉 Git ──> PA1: 实现表达式求值 & 简单调试器 (Monitor)
                                                    │
  PA4: 分时多任务与虚存管理 (运行仙剑) <── PA3: 批处理系统 (异常控制流与文件系统) <── PA2: 汇编指令译码与输入输出
```

*   **PA0：工欲善其事 (环境准备)**
    *   **目标**：安装并配置 Ubuntu Linux 开发环境，熟悉命令行操作，配置好 GCC/GDB/Make/Git 工具链。
*   **PA1：Monitor (基础设施建设)**
    *   **目标**：在 NEMU 中实现一个简易调试器。用 C 语言手写一个表达式求值解析器，实现单步执行、打印寄存器、扫描内存和设置监视点（Watchpoint）功能。
*   **PA2：指令译码与输入输出 (硬件模拟的核心)**
    *   **目标**：实现 CPU 的取指、译码、执行循环。用 C 语言模拟汇编指令对寄存器和内存的修改。模拟简单的键盘、VGA 显卡和时钟，让 NEMU 能够输出图形。
*   **PA3：批处理系统 (穿越时空的旅程)**
    *   **目标**：从冯·诺依曼计算机系统演进为最简单的操作系统。在 Abstract-Machine 的支持下，引入异常控制流（ECF），实现简易的系统调用（Syscalls）与基本的文件系统。
*   **PA4：分时多任务与虚存 (虚实交错的魔法)**
    *   **目标**：将批处理系统演化为多道程序/分时多任务系统。实现支持虚拟页式存储管理，增加抢占式调度与外部中断处理，并在 NEMU 上成功运行分时多任务与《仙剑奇侠传》。

## 💡 必考复习考点与实验攻略

1.  **RTFM (Read The F**king Manual)**：
    *   这是 PA 指导书上出现频率最高的词。PA 指导书里隐藏了大量的技术细节和实现提示。遇到 Bug，90% 的原因都是因为你漏掉了指导书中的某一句关键描述。
2.  **熟练使用 GDB 调试**：
    *   PA 代码量巨大（数千行），且涉及指针和内存的直接操作。学会使用 GDB 设置断点、查看内存、跟踪调用栈是你的生存保障。**绝对不要用 `printf` 打印大法去调试 PA**。
3.  **理解状态机 (State Machine)**：
    *   计算机是一个确定的状态机。程序在 NEMU 里跑崩溃了，说明在某一步状态转换（如指令执行、寄存器修改）中发生了预期外的改变。通过比对你的模拟器输出的客户程序执行流与标准执行流（如 GDB/QEMU），可以快速定位导致状态异常的那条指令。
