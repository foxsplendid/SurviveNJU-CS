# 软件工程

*   **课程学分**：2 学分
*   **授课学期**：大三秋季学期
*   **授课教师**：计科系教师组
*   **修读阶段**：计算机系必修课

---

## 🌟 课程简介与地位

相较于纯粹的理论算法或系统底层，《软件工程》是一门关注**“如何以工程化的方法去开发和维护大型软件系统”**的课程。本课程的目标是帮助你从“单兵作战写作业”的思维，转变为“团队协同开发符合工业规范的商业级软件”的工程思维。

这门课介绍了软件开发生命周期中的需求分析、系统设计、项目管理、设计模式以及软件测试等核心方法。

---

## 📚 核心内容与实验

### 1. 核心理论知识点
*   **软件开发模型**：瀑布模型、增量模型、敏捷开发（Agile - Scrum 框架）。
*   **需求分析**：用例图（Use Case Diagram）、需求规格说明书。
*   **系统设计与 UML**：
    *   **类图（Class Diagram）**（高频考点）：掌握类与类之间的关系（关联、聚合、组合、继承、实现）。
    *   时序图（Sequence Diagram）、状态机图。
*   **设计模式 (Design Patterns)**（重难点）：
    *   创建型模式：单例模式、工厂模式。
    *   结构型模式：适配器模式、装饰器模式、代理模式。
    *   行为型模式：观察者模式、策略模式、模板方法。
*   **软件测试**：黑盒测试（等价类划分、边界值分析）、白盒测试（路径覆盖、逻辑覆盖）。

### 2. 课程大作业与实验
通常要求 3-5 人组队，合作开发一个完整的软件系统（如在线商城、协作办公系统等）。在过程中体验敏捷开发的迭代过程，撰写需求说明书，使用 Git 进行代码版本控制，并进行单元测试。

## 💡 备考与学习攻略

1.  **攻克设计模式与 UML 类图**：
    *   期末试卷中，**设计模式的改写和类图绘制**是必考大题。题目通常会给出一个设计极其臃肿、扩展性极差的代码片段，要求你指出其违背了哪些软件设计原则（如单一职责原则 SRP、开闭原则 OCP 等），并选择合适的设计模式对其进行重构，画出重构后的类图。
    *   平时学习时，多动手用笔画画类图，理清虚线、实线、空心箭头、实心箭头的区别。
2.  **合理协调团队分工**：
    *   大作业极度考验团队合作。队伍中如果出现“搭便车”的同学，会导致整个项目的进度崩盘。在大作业启动初期，就要明确分工，规范使用 Git 分支（如使用 Feature Branch 工作流），避免最后一天强行合并代码时发生海量冲突。
3.  **与软件学院课程的对比**：
    *   计算机系的《软件工程》偏向理论和广度的介绍。如果你想学习更深、更硬核的工程开发和静态代码分析，可以跨院系参考 **[软件学院核心课程](../se-core/index.md)** 的资料。
