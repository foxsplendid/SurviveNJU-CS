## 🌟 课程简介与地位

《编译原理》是计算机专业“系统能力培养线”的集大成课程之一。本课程回答了另一个底层核心问题：

> **“高级语言编写的纯文本代码，是如何被翻译成计算机硬件能够直接执行的二进制机器码的？”**

这门课不仅是自动机理论、形式语言理论的直接应用，更是一门实践性极强的工程课。你将在本课程中学习到词法分析、语法分析、语义分析、中间代码生成、代码优化以及目标代码生成的全过程。

---

## 🛠️ 课程大作业：编译器的实现

南大编译课的标志性大作业是**手写或使用工具实现一个编译器**（通常是把类 C 语言编译为 MIPS/RISC-V 汇编代码，或者是自定义的中间代码）。

### 🛠️ 编译器的开发阶段
1.  **词法分析 (Lexical Analysis)**：使用 Flex 或手写自动机，将纯文本代码流切分为一个一个的 Token（如关键字、标识符、运算符）。
2.  **语法分析 (Syntax Analysis)**：使用 Bison 或手写递归下降解析器，将 Token 流解析为一棵抽象语法树 (AST)。
3.  **语义分析 (Semantic Analysis)**：进行类型检查、符号表构建，确保程序符合语义规则（如变量先声明后使用、赋值类型匹配）。
4.  **中间代码生成 (IR Generation)**：将 AST 转换为便于优化和跨平台的三地址码 (Three-Address Code, 3AC) 等中间表示。
5.  **目标代码生成 (Code Generation)**：进行寄存器分配，将中间代码翻译为目标机器的汇编语言。

## 💡 备考与学习攻略

1.  **熟练掌握形式语言与自动机**：
    *   期末考试理论部分包含大量推导题。如：**正规式转 NFA、NFA 确定化为 DFA、DFA 的最小化**。
    *   掌握 **LL(1) 预测分析表** 和 **LR(1)/LALR(1) 分析表** 的构建过程。这是考卷中的大题高发区，公式繁多，容易出错，必须通过课后习题进行巩固。
2.  **掌握语法制导翻译 (SDT) 与翻译模式**：
    *   理解如何通过在语法规则中嵌入语义动作，实现求值、类型检查或代码生成。
3.  **合理分工大作业**：
    *   大作业通常为两人合作或个人独立完成。由于编译器的每个阶段都极度依赖前一个阶段的输出，如果前期的词法/语法分析留有 Bug，后期的语义分析和代码生成将痛苦万分。所以每个阶段务必编写充分的测试用例进行单元测试。
