# 计算机网络

## 知识地图

- **应用层**：HTTP、DNS、电子邮件及应用协议设计。
- **传输层**：UDP、TCP、可靠传输、流量控制和拥塞控制。
- **网络层**：IP、CIDR、转发、路由算法与域内/域间路由。
- **链路层**：以太网、ARP、交换、差错检测和共享介质访问。
- **网络安全与演进**：TLS、NAT、IPv6、无线网络等主题是否展开取决于课程大纲。

“三次握手”“拥塞窗口”或某类路由计算可能是重要练习，但在没有当年试卷说明时不能标为固定必考。

## 实践方式

Wireshark 抓包、Socket 编程、网络模拟和协议栈实现都是常见学习方式，但不代表当前教学班的固定实验。使用抓包数据时注意隐私和授权，不采集无关用户的流量。

可自行进行的低风险练习包括：

1. 对自己发起的 HTTP/DNS 请求记录分层报文和时序；
2. 编写最小 TCP/UDP 客户端与服务器，观察分包、重传和关闭行为；
3. 用受控拓扑推演最长前缀匹配、距离向量和链路状态算法；
4. 对计算结果标明 RTT、带宽、窗口和丢包模型等假设。

## 学习方法

1. 始终区分协议提供的服务、报文格式、状态机和实现策略。
2. 画时序图时标出方向、序号、确认号、状态与超时事件。
3. 计算吞吐或利用率前先统一单位，并说明是否忽略头部、排队和处理时延。
4. 结合抓包验证教材模型，同时记录加密、缓存和中间设备带来的偏差。
