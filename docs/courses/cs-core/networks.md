# 计算机网络

*   **课程学分**：4 学分
*   **授课学期**：大三春季学期
*   **授课教师**：计科系教师组
*   **修读阶段**：计算机系必修课

---

## 🌟 课程简介与地位

《计算机网络》是连接理论与互联网现实世界的桥梁课。本课程采用经典的**“自顶向下”（Top-Down Approach）**方法，从应用层、传输层、网络层、链路层到物理层，层层拆解复杂的互联网协议栈。

你会在这门课中明白：浏览器输入 URL 后发生了什么、TCP 是如何保证可靠传输的、路由器是如何在茫茫网络中为你寻找最短传输路径的。

---

## 📚 核心内容与实验

### 1. 核心理论考点
*   **应用层**：HTTP/HTTPS 协议、DNS 解析过程、SMTP/POP3 邮件协议。
*   **传输层**：**TCP 三次握手与四次挥手**（必考）、可靠数据传输原理 (GBN, SR)、**TCP 拥塞控制与流量控制**（拥塞窗口变化折线图分析，必考）。
*   **网络层**：IP 地址划分与 CIDR、路由器工作原理、路由选择算法（距离向量算法 DV, 链路状态算法 LS - Dijkstra）、BGP/OSPF/RIP 协议。
*   **链路层**：ARP 协议、以太网协议、CSMA/CD 与 CSMA/CA 冲突避免机制。

### 2. 实验要求
*   **Wireshark 抓包实战**：通过抓取真实的 HTTP/TCP/IP 数据包，分析其首部结构、握手过程和分片策略，将抽象的协议转换为具象的字节流。
*   **Socket 编程**：用 C++/Python/Java 编写经典的多线程 TCP/UDP 回显服务器与客户端，理解网络 I/O 编程的基本模型。

## 💡 备考与学习攻略

1.  **熟记协议交互图**：
    *   期末考试中，画出 TCP 连接建立/释放的时序图、或者计算滑动窗口协议下的信道利用率是经典题型。
    *   在草稿纸上多画几次三次握手和四次挥手的状态转移图（SYN_SENT, ESTABLISHED, FIN_WAIT_1, TIME_WAIT 等）。
2.  **网络层计算题专项**：
    *   熟练掌握子网划分与路由聚合（最长前缀匹配）。
    *   搞懂距离向量算法（DV）中的“坏消息传得慢”（Count-to-Infinity）问题及水平分割解决方法。
3.  **大厂面试的高频区**：
    *   计算机网络是各大互联网公司开发岗位面试的**最重灾区**。TCP 可靠性传输机制、拥塞控制、三次握手四次挥手、HTTPS 握手流程，几乎是面试官必问的八股文。建议在修读这门课时就将这些概念彻底弄懂，而不是考试前死记硬背。

## 🔗 推荐资源

*   📖 **核心教材**：《计算机网络：自顶向下方法》（Computer Networking: A Top-Down Approach, James F. Kurose 著），讲解非常生动，配图极其精美，强烈推荐阅读中文或英文原版书。
*   🎥 **公开课推荐：Stanford CS144**：斯坦福大学的计算机网络公开课，其实验要求手写一个功能完备的 TCP 协议栈（包含 IP 分片与重组、滑动窗口等），适合想硬核挑战的同学自学。
