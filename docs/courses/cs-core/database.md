# 数据库系统

!!! info "课程信息需按年份核对"
    本页不固定学分、学期、教师、培养性质、实验技术栈或考试题型。不同教学班对数据库设计、查询处理和系统实现的侧重可能不同。

## 核心内容

- **关系模型与关系代数**：选择、投影、连接及查询表达。
- **SQL**：查询、聚合、子查询、约束、视图和数据更新。
- **数据库设计**：E-R 建模、函数依赖、候选键、范式、无损连接与依赖保持。
- **事务**：ACID、可串行化、恢复和隔离级别。
- **并发控制**：锁、时间戳、MVCC 等机制及其适用条件。
- **存储与索引**：页式存储、B+ 树、哈希索引和缓冲管理。
- **查询处理**：逻辑/物理计划、连接算法和代价估计。

以上是数据库课程常见知识地图，不表示每一项都是当前教学班的固定考试范围。

## 实践形态

实践可能是 SQL 与模式设计、小型应用、数据库内核组件或查询优化实验。MySQL、PostgreSQL 及前端/后端框架只是可能选择，不应写成统一要求。开始项目前确认指定 DBMS、版本、数据集、允许使用的库和验收方式。

## 学习方法

1. 对模式设计写清函数依赖、候选键和分解目标，不只套用范式名称。
2. 用优先图等方法分析调度，并区分冲突可串行化、恢复性和隔离现象。
3. 对 SQL 同时验证结果正确性、NULL 语义、重复行和执行计划。
4. 将 B+ 树、锁和日志机制放在具体故障或性能场景中理解。
