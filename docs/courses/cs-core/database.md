# 数据库系统

*   **课程学分**：3 学分
*   **授课学期**：大二春季学期
*   **授课教师**：计科系教师组
*   **修读阶段**：计算机系必修课

---

## 🌟 课程简介与地位

数据是现代软件系统的核心资产。如何在保证安全、高效、一致的前提下存储和管理海量数据，是《数据库系统》要解决的核心问题。本课程分为**应用层面**（如何写好 SQL、设计合理的数据库模式）与**系统级底层层面**（数据库内核中的事务并发控制、索引结构与查询优化）。

---

## 📚 核心内容与期末考点

### 1. 核心理论考点
*   **关系模型与关系代数**：使用投影、选择、连接等代数运算来表示数据查询操作（期末大题常考）。
*   **SQL 语言**：熟练编写复杂的 SELECT 查询（多表连接、分组 GROUP BY、子查询、聚合函数）。
*   **数据库设计与规范化理论**：
    *   **E-R 图设计**：将实体关系模型转换为关系模式。
    *   **函数依赖与范式（Normal Forms）**（必考大题）：掌握 **1NF, 2NF, 3NF, BCNF** 的定义，并能够手推判断一个关系模式属于第几范式、以及如何进行无损连接和保持依赖的范式分解。
*   **事务与并发控制**：
    *   事务的 **ACID 属性**（原子性、一致性、隔离性、持久性）。
    *   **并发控制协议**（必考）：两段锁协议 (2PL, Two-Phase Locking)、死锁检测与预防、封锁粒度。
*   **系统内核底层**：B+ 树索引结构、查询优化（关系代数等价变换）。

### 2. 课程大作业与实验
通常要求实现一个完整的软件系统（如教务管理系统、图书租赁系统等），采用典型的后端架构，使用 MySQL 或 PostgreSQL，手写 SQL 语句，并在前端展示。

---

## 💡 备考与学习攻略

1.  **攻克“模式分解”大题**：
    *   范式判定和范式分解是期末试卷的“送分题”也是“送命题”。熟练掌握求属性集闭包（Attribute Closure）、求候选键（Candidate Key）的算法。
    *   多做几道往年真题中的范式分解题目，熟悉无损连接性测试（画表法）的步骤。
2.  **注重并发调度分析**：
    *   掌握冲突可串行化（Conflict Serializability）的判定方法，学会构建优先图（Precedence Graph）并判断是否存在环。
3.  **大厂求职高频区**：
    *   在后端开发和系统开发面试中，数据库是绝对的重灾区。**B+ 树索引的原理、隔离级别与脏读/幻读/不可重复读、悲观锁/乐观锁、MVCC 机制、Redis 缓存设计**是常考的核心内容。在学完这门课后，建议结合求职八股文进行查漏补缺。
