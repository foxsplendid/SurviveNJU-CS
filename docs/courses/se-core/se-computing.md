# 软件工程与计算

*   **课程学分**：4 学分（每阶段）
*   **授课学期**：大一春季学期 (I) / 大二秋季学期 (II) / 大二春季学期 (III)
*   **授课教师**：软件学院课程组
*   **修读阶段**：软件学院必修课

---

## 🌟 课程简介与地位

《软件工程与计算》（Software Engineering and Computing, 简称 SEC）系列课程是**南京大学软件学院的“脊梁”与标志性大课**。它将传统的软件工程理论（如开发模型、设计模式、软件架构）与强度的动手实践（大作业项目开发）完美结合。

其中，最让软院学子魂牵梦绕的是大二下的 **《软件工程与计算 II》**。在这门课中，你需要与 5-6 位同学自组团队，在四个月内从零开发一个完整的中型软件系统（通常是一个包含前后端交互的 Web/Mobile 系统，如酒店预订系统、低代码平台或在线协作平台）。

---

## 🏗️ SEC 阶段演进与实验大作业要求

### 1. 软件工程与计算 I（个人单兵作战）
*   **目标**：掌握 Java 语言基础，熟悉面向对象编程思想（OOP），学会使用简单的单元测试（JUnit）。

### 2. 软件工程与计算 II（中型团队大作业 - “炼狱模式”）
*   **目标**：5-6 人组队，经历软件生命周期的全过程：
    *   **需求阶段**：产出《需求规格说明书》（SRS），画出用例图和系统顺序图。
    *   **架构设计**：采用三层架构（展示层、业务逻辑层、数据层）或微服务架构，产出《体系结构设计说明书》。
    *   **详细设计**：产出《详细设计说明书》，绘制时序图和状态机图。
    *   **编码与测试**：手写单元测试、集成测试，并在 Jenkins/GitHub Actions 上配置持续集成流。

### 3. 软件工程与计算 III（系统性能与大规模系统）
*   **目标**：关注大规模分布式系统的架构设计、缓存设计、性能调优和部署运维。

## 💡 备考与学习攻略

1.  **攻克“Git 版本冲突”与“合并灾难”**：
    *   团队协作开发最常遇到的就是代码冲突（Merge Conflict）。一定要在大作业启动之初就制定严密的 Git 协作规范：
        *   禁止在 `main` 或 `master` 分支直接写代码。
        *   每个人在独立的 `feature/xxx` 分支上开发，经过 Pull Request (PR) 和代码审查（Code Review）后合并。
        *   勤拉取（`git pull`），减少代码分叉造成的冲突。
2.  **别把文档当摆设**：
    *   SEC 课程对文档规范度的要求极高。很多队伍的代码写得很漂亮，但因为需求规格说明书、详细设计说明书格式混乱、缺乏追溯关系而拿了低分。请使用标准的 Markdown/Word 模板认真撰写需求文档和架构文档。
3.  **单元测试覆盖率（Test Coverage）**：
    *   大作业的最终评审通常会对单元测试的覆盖率（如指令覆盖率、分支覆盖率）提出硬性指标（通常要求达到 75%+）。在开发业务逻辑时，同步编写 JUnit 测试用例，不要等到最后一晚去“凑”测试代码。
