# 软件测试

*   **课程学分**：3 学分
*   **授课学期**：大三春季学期
*   **授课教师**：软件学院教师组
*   **修读阶段**：软件学院必修课 / 计算机系选修课

---

## 🌟 课程简介与地位

如果说《软件分析》是通过静态证明来确保代码没有 Bug，那么《软件测试》（Software Testing）则是通过**动态运行、输入构造、测试覆盖率分析以及自动化测试技术**来验证软件行为是否符合预期。

本课程涵盖了传统的黑盒测试用例设计、白盒测试覆盖率理论，并重点探讨了现代自动化测试前沿方向，如变异测试（Mutation Testing）、符号执行（Symbolic Execution）和广泛应用于网络协议与操作系统安全领域的**模糊测试（Fuzzing）**。

---

## 📚 核心内容与期末考点

### 1. 核心理论知识点
*   **黑盒测试方法**：等价类划分法（Equivalence Partitioning）、边界值分析法（Boundary Value Analysis）、因果图与判定表法。
*   **白盒测试覆盖率准则**（高频考点，必考计算）：
    *   语句覆盖、判定/分支覆盖、条件覆盖。
    *   **判定-条件覆盖 (MCDC, Modified Condition/Decision Coverage)**：如何为复杂布尔表达式构造最简测试用例对，证明每个独立条件都能独立影响整个判定结果。
*   **控制流与路径测试**：圈复杂度（Cyclomatic Complexity）计算公式：$V(G) = E - N + 2P$。
*   **变异测试 (Mutation Testing)**：变异算子设计、变异得分计算（如何杀死变异体）。
*   **自动化测试前沿**：
    *   **模糊测试 (Fuzzing)**：基于变异和基于生成的 Fuzzing 流程；AFL 工具的基本原理（插桩技术指导的反馈机制）。
    *   **符号执行 (Symbolic Execution)**：符号值替换、路径约束求解（SMT Solver）。

### 2. 课程大作业与实验
通常包含以下自动化测试实践：
*   **白盒测试实战**：使用 JUnit/JaCoCo 针对复杂 Java 模块编写单元测试，手动设计满足 MC/DC 准则的测试用例。
*   **模糊测试 (Fuzzing) 实验**：使用 AFL 或 LibFuzzer 对一个含有安全漏洞的 C 程序（如图像解析器、压缩包解析器）进行模糊测试，捕获并分析 Crash 堆栈，找出漏洞成因。

## 💡 备考与学习攻略

1.  **熟练手算 MC/DC 测试用例**：
    *   期末大题常考对于一个复杂的布尔判定（如 $A \land (B \lor C)$），要求你写出能够实现 MC/DC 覆盖的最小测试用例集合，并详细列出每个条件的独立测试用例对。
    *   复习时，梳理真值表，掌握通过 XOR 运算推导独立条件的方法。
2.  **圈复杂度与基路径测试**：
    *   根据给定的代码绘制控制流图 (CFG)，计算圈复杂度，并写出基路径集合。
3.  **大作业注重报告质量**：
    *   在写 Fuzzing 实验大作业时，不仅要把漏洞 Fuzz 出来，更要写清楚你的 Fuzz 策略、变异种子设计、执行效率调优（Exec/sec）以及最终 Crash 的成因分析。
