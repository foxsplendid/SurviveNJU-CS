# 软件学院课程与软件质量专题

!!! info "目录归档不等于课程归属"
    本目录同时收录软件学院培养中的工程课程和跨院系的软件质量公开资源。页面所在目录不能证明课程是软件学院必修，也不能固定学分、学期或教师；正式信息应以本人适用的培养方案和教务系统为准。

## 内容分组

- [软件工程与计算](se-computing.md)：软件学院 SEC 系列的能力地图。课程阶段、团队规模、项目技术栈和验收指标按当年任务书核对。
- [软件测试](software-testing.md)：测试设计、覆盖准则与自动化缺陷发现的通用知识地图；具体实验工具和考核范围不作固定承诺。
- [软件分析](software-analysis.md)：PASCAL 团队公开的静态分析与 Tai-e 资源。PASCAL 教学页将团队关联到计算机科学与技术系，本页按主题收录，不将其误写为软件学院固定必修。

## 核验入口

- [南京大学软件学院本科培养方案](https://software.nju.edu.cn/rcpy/bks/20260514/i388724.html)：先选择本人适用版本，再确认课程名称和培养性质。
- [PASCAL Research Group](https://pascal-lab.net/) 与 [Tai-e 教学主页](https://tai-e.pascal-lab.net/)：分别核对研究组归属、当前公开项目和软件分析教学资源。
- 各课程当年教学平台：用于确认教师、分组、实验、评分和学术诚信要求，优先级高于本指南及学生经验帖。

## 学习建议

1. 团队项目开始前明确需求、接口、分支策略、代码审查和验收证据。
2. 把测试、静态检查和文档追踪纳入日常迭代，不在截止前集中补齐。
3. 不使用往届源码、报告和项目模板作为当前作业起点；只采用课程组明确许可的材料。
4. 对第三方课程评价保留年份与来源，不把个人难度、工作量或成绩经验写成普遍事实。
