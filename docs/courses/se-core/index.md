# 软件学院核心课程

南京大学软件学院成立于 2002 年，是教育部首批批准设立的国家示范性软件学院。学院的培养理念聚焦于**“工程化、团队化、实践化”**。与偏向算法理论和底层系统的计算机系不同，软件学院更强调大型复杂软件系统的架构设计、全生命周期开发管理、团队工程协作以及静态代码分析与测试技术。

本板块汇总了软件学院核心专业课的攻略和学习指南。

---

## 🏗️ 软件学院核心培养路线

软件学院的核心课程高度聚焦于软件工程实践：

```
  大一至大二（工程基础与团队协作）
  程序设计基础 ──> 软件工程与计算 I ──> 软件工程与计算 II (中型团队大作业) ──> 软件工程与计算 III
                                                                           │
  大三阶段（系统质量与前沿工程）                                             │
  软件分析 (静态分析 Tai-e) <──────────────────────────────────────────────┘
  软件测试 / 软件体系结构
```

*   **大一至大二：软件工程与计算 (SEC) 系列**。这是软院的脊梁课，大二下的《软件工程与计算 II》要求 5-6 人组队进行为期一个学期的大型项目开发，从需求、设计、测试到持续集成（CI/CD），直接体验工业界的真实开发流。
*   **大三：软件质量保证**。以《软件分析》和《软件测试》为代表，教你如何利用静态分析工具（如南大自主开发的 **Tai-e**）和动态测试框架去保证大型软件的安全性、可靠性和性能。

## 💡 给软件学院新生的学习建议

1.  **打牢 Git 与版本控制基础**：
    *   软院的大作业几乎全是团队协作。如果在团队开发中不会解决 Git 冲突、分不清主分支与特征分支，大作业的协作将会是一场灾难。大一入学就要熟练掌握命令行 Git 的常用操作。
2.  **注重软件设计原则，而非单纯敲代码**：
    *   不要当只懂得堆砌业务代码的“码农”。在学 SEC II 和设计模式时，理解开闭原则、单一职责原则以及接口分离原则。良好的软件架构设计能省去你后期重构时 90% 的痛苦。
3.  **积极利用开源工具链**：
    *   在开发过程中，学会配置 CI/CD 自动构建部署（如 GitHub Actions）、手写 JUnit 单元测试用例。这些工程化素养不仅能帮你在大作业中拿高分，也是求职面试时的巨大加分项。
