# 正确的学习思维

「大学的学习方式和高中完全不同。如果你还试图用高中的‘题海战术’或‘被动接受’来应对南大泛CS的课程，很快你就会感到力不从心。」

作为学长学姐，我们见过太多优秀的同学在入学后因为学习思维没有转变而陷入痛苦。在这篇文章中，我们想和你聊聊在南大CS求学最核心的几种学习思维。

---

## 🔄 从高考思维到大学思维的转变

高考的本质是**“在一张已知考纲的试卷上拿尽可能高的分数”**，其特点是信息完全对称、有标准答案、靠反复练习（刷题）即可提升。

而大学，尤其是计算机专业的本质是**“在高度信息不对称、技术迭代极快、没有标准答案的环境下解决实际问题”**。

```
高中思维：上课听讲 -> 课后刷题 -> 考试拿100分 -> 终点
大学思维：面对陌生需求 -> 查阅文档/搜索 -> 编写代码 -> 调试排错 -> 解决问题
```

*   **变被动为主动**：在南大，没有班主任会每天督促你。老师在课堂上可能只讲最核心的理论框架，具体的实验实现、API的使用，都需要你课后主动查阅文档和搜索引擎去自学。
*   **接受“不确定性”**：在系统课程（如 ICS PA、OS Lab）中，你拿到的实验说明书可能长达几十页，且包含故意留给你的探索空间。没有一步一步的“保姆教程”，你需要学会在迷茫中摸索，学会通过阅读源码来寻找答案。

---

## ⚖️ 正确对待 GPA：是入场券，但不是唯一

南京大学的 GPA 计算非常严格。我们必须承认，绩点非常重要，它是你保研、申请出国、评选奖学金的直接依据。但请务必记住：**GPA 是你通往下一阶段的入场券，但绝对不是你的全部身价。**

*   **不要为绩点“买单”**：不要为了追求 4.7+ 的绩点而只选水课，避开那些真正硬核但可能给分稍低的王牌课程（如 jyy 老师的操作系统、魏恒峰老师的编译原理）。这些硬核课程带给你的底层系统能力和思维训练，将在你未来的科研、面试、工作中成为你最宽的护城河。
*   **无意义的“内卷”不可取**：在作业中为了迎合评分细则做大量毫无意义的排版工作、或者通过作弊抄袭来获取高分，都是在浪费时间。计算机是一个“用代码说话”的行业，你的真实代码能力和解决问题的水平，面试官用 5 分钟的机试和提问就能测出来。

---

## 💻 实践与系统思维：用代码说话

南京大学计算机系在全国最著名的特色之一就是**对“计算机系统能力”的极高要求**。这离不开蒋炎岩（jyy）老师等提倡的教学理念：

!!! quote "蒋炎岩老师公开讲义观点"
    “程序是状态机。计算机系统是没有魔法的，所有看似神奇的表现，背后都是一行行确定执行的代码和状态转换。”

*   **Training > Learning**：计算机不是看书看懂的，是写代码、调 Bug 练出来的。在 ICS 课程中，你需要从零实现一个 NEMU 模拟器。在这条路上，你会遇到无数个 segmentation fault 和死循环。每一次查阅 GDB 调出 bug，都是你系统思维（System Mindset）建立的过程。
*   **拒绝当“调包侠”**：不要一遇到问题就想着去网上调一个开源库或者用 AI 帮你写完。在本科阶段，花时间去理解底层的进程调度、内存分配、编译优化，能够让你在未来面对复杂系统故障时，具有别人不具备的深度。

---

## 🛡️ 反卷与心态调整

在南大泛CS专业，你身边会环绕着各路“大佬”：有高中就拿过 NOIP 一等奖的竞赛生，有一入学就自学完大二课程的技术极客。在这种环境下，很容易产生同辈压力（Peer Pressure）。

*   **只与昨天的自己对比**：每个人的起点和兴趣点不同。有的人擅长算法竞赛，有的人擅长工程开发，有的人适合做理论研究。找到你自己的节奏，只要你每天都在比昨天多写几行有用的代码、多读懂一篇论文，你就是在前进。
*   **积极的心理暗示**：计算机专业课程多、实验重，偶尔的挫折（比如实验过不了评测机，期末考试没考好）是常态。这并不代表你不适合学计算机，这只是学习过程中的一关，咬牙走过去，视野就会开阔。
