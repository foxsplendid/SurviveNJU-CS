# 关于本书

本指南是一份由社区共同维护的南京大学泛 CS 非官方学习与校园生存指南。它整理官方信息入口、带适用范围的课程与发展经验、公开资源索引和校园求助路径，目标是降低检索与交叉核验成本。

---

## 诞生背景

故事的起点，是上海交通大学那本著名的 [SurviveSJTU 手册](https://survivesjtu.gitbook.io/survivesjtumanual/)。

这类学生共建手册把分散的校园经验、办事路径和选择依据组织为可持续维护的文档。它让我们意识到：**大学里稀缺的不只是知识本身，也包括“从哪里核验信息”以及“如何在具体条件下做出选择”。**

南京大学泛 CS 方向同样存在大量分散信息：不同年级课程如何对应、哪些规则只适用于某一学期、如何寻找研究组公开入口、遇到学籍或校园问题应联系哪里。若不记录来源和适用条件，经验很容易在传播中变成失真的“统一规则”。

SurviveNJU-CS 就是为了降低这种信息差而生的。我们受 SurviveSJTU 启发，借鉴了 [CS自学指南](https://csdiy.wiki/) 等前辈资源的组织方式，希望把南大 CS 圈子里的集体经验沉淀下来，形成一份可以持续更新的活文档。

## 本指南的定位

!!! note "请先看这里"
    本指南是**非官方编纂**，不代表南京大学或任何院系、部门、课程组的立场。事实性内容按来源核验，经验与观点应单独标注适用条件；两者都不能替代适用于本人的正式文件、教务系统记录或主管部门答复。

**我们是什么：**

- 事实入口的**整理与交叉核验**：优先链接学校、院系、教师课程页和项目维护者的一手页面，并记录版本、日期与冲突
- 经验内容的**条件化沉淀**：说明院系、年级、学期和样本边界，不把个人经历包装成普遍结论
- 资源链接的**索引与导航**：不重新发布受版权保护的内容，只整理公开入口、授权状态和使用风险
- 校园任务的**路径地图**：帮助定位学业程序、医疗心理、紧急求助和公共服务入口

**我们不是什么：**

- 不是官方教学大纲或培养方案的替代品
- 不是应试捷径手册（我们鼓励真正的学习）
- 不是第三方资源授权状态的背书
- 不是对第三方工具安全性、课程收益或升学就业结果的保证
- 不是永远有效的标准答案；时效性事实必须在使用时重新核验

## 覆盖范围

本指南当前以南京大学校级学院列表和公开课程入口为边界，收录以下六个泛 CS 相关学院。覆盖深度并不相同，表格不会把“已有学院入口”写成“课程百科已经完整”。

| 学院 | 当前覆盖状态 |
|---|---|
| 计算机学院 | 已有系统、算法、网络、编译等课程知识地图与公开课程入口 |
| 软件学院 | 已有软件工程与计算、软件分析、软件测试等课程与资源边界说明 |
| 人工智能学院 | 已有数学基础、机器学习、深度学习、NLP、CV 等带学期开设记录或知识地图 |
| 智能科学与技术学院 | 已收录学院官网与带学期课表入口；尚未确认公开、完整、现行的培养方案正文 |
| 智能软件与工程学院 | 已收录学院官网与带学期课表入口；尚未确认公开、完整、现行的培养方案正文 |
| 机器人与自动化学院 | 已收录学院、课程体系栏目与带学期课表入口；不从栏目名称推断培养要求 |

!!! tip "非CS专业也欢迎"
    其他学院、辅修或跨专业学习者也可以使用通用知识地图、资源索引和校园服务入口，但课程性质、学分与毕业要求必须回到本人培养方案核对。

## 如何使用本指南

本指南不需要从头到尾线性阅读——它更像是一本**随用随查的工具书**。

推荐使用方式：

1. **刚入学时**：从「新生入门」和「校园办事」定位本人院系、培养方案与常用官方入口
2. **选课前**：查阅「课程指南」，先确认具体开设身份，再使用知识地图和公开资源
3. **办理学业事项时**：从「学业规则」找到现行制度与当学期通知，不套用旧流程日期
4. **面临升学或就业决策时**：区分学校规则、目标单位要求和个人经验，不依赖单一时间线
5. **遇到紧急情况时**：直接查看「紧急情况与校园求助」，优先联系相应公共应急服务

## 内容更新说明

大学政策、课程设置、联系方式和申请安排会变化。本指南尽力在易变事实旁标注适用年份、学期或核验日期，但页面级日期不代表其中每个外部来源都在同一天发布，也不保证内容永久有效。

!!! warning "以官方信息为准"
    涉及学分、培养方案、学籍、申请截止日期、医疗或安全等重要事项，请以适用于本人的学校部门、学院、课程组和教务系统当前信息为准；官方页面互相冲突时，应向发布或主管部门确认。

### 如何参与更新

本指南托管在 GitHub，使用 MkDocs Material 构建，任何人都可以通过 Pull Request 来修正错误、补充内容、添加新章节。

- 发现内容已过时？→ 提一个 [Issue](https://github.com/foxsplendid/SurviveNJU-CS/issues) 并附上页面位置与可核验来源
- 有新内容想贡献？→ 参考 [贡献指南](./contributing.md) 提交 PR
- 不熟悉 Git？→ 可以先提交最小化 Issue；Issue 是公开页面，请勿粘贴个人联系方式、身份材料、成绩单或未公开课程内容

我们相信，一份指南的生命力来自于持续的社区贡献。每一位曾经在南大读书的同学，都是这本书潜在的共同作者。
