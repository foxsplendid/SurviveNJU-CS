# 关于本书

本指南是由南京大学泛CS方向同学自发组织、共同编写的一份非官方学习与生存手册。你手上看到的这些文字，来自那些在仙林园区走过四年（乃至更多年）的普通学生——他们经历过同样的迷茫，也摸索出了一些有用的方法，想把这些留给后来者。

---

## 诞生背景

故事的起点，是上海交通大学那本著名的 [SurviveSJTU 手册](https://survivesjtu.gitbook.io/survivesjtumanual/)。

那本书以坦诚到近乎刺耳的笔调，讲述了 SJTU 学生在大学四年中真实面对的困境与选择，迅速在工科学生圈子里广为流传。它让我们意识到：**大学里最稀缺的资源，往往不是知识本身，而是"如何获取知识"以及"如何做出选择"的信息。**

南京大学的 CS 方向同样有丰富的"隐性知识"在学生之间口耳相传——哪门课应该早点选、哪位老师的科研方向更适合保研、夏令营套磁邮件该怎么写……但这些经验往往随着一届届学生毕业而流失，下一届人只能重新踩同样的坑。

SurviveNJU-CS 就是为了打破这种信息孤岛而生的。我们受 SurviveSJTU 启发，借鉴了 [CS自学指南](https://csdiy.wiki/)、[NJU Box](https://box.nju.edu.cn/) 等众多前辈资源的思路，希望把南大 CS 圈子里的集体经验沉淀下来，形成一份可以持续更新的活文档。

## 本指南的定位

!!! note "请先看这里"
    本指南**不是官方文件**，不代表南京大学计算机科学与技术系、人工智能学院、软件学院或任何官方机构的立场。所有内容均为学生个人经验与观点，仅供参考。

**我们是什么：**

- 学生经验的**聚合与传承**：把分散在各个群聊、朋友圈、个人博客里的有价值经验整理到一个地方
- 资源链接的**索引与导航**：不重新发布受版权保护的内容，而是帮你找到正确的地方去获取它们
- 选择信息的**透明化**：让每个南大 CS 学生都能在做重大决策时，获得足够充分的参考信息

**我们不是什么：**

- 不是官方教学大纲或培养方案的替代品
- 不是应试捷径手册（我们鼓励真正的学习）
- 不是永远正确的标准答案（大学里很少有标准答案）

## 如何使用本指南

本指南不需要从头到尾线性阅读——它更像是一本**随用随查的工具书**。

推荐使用方式：

1. **刚入学时**：先读完「新生入学指南」，建立对大学生活的基本认知
2. **选课前**：查阅「课程资源」中对应课程的评价与资料
3. **面临升学决策时**：阅读「保研/考研」或「出国申请」对应章节，梳理时间线
4. **需要具体帮助时**：利用左侧导航栏或顶部搜索框直接定位

每个页面顶部都有简短的"本页导读"，可以帮助你快速判断这个页面是否对你有用。

## 内容更新说明

大学政策、课程设置、申请行情每年都在变化。本指南的内容存在**时效性限制**，我们尽力标注每个章节的最后更新日期，但无法保证所有内容随时与最新情况同步。

!!! warning "以官方信息为准"
    涉及学分要求、培养方案、申请截止日期等重要信息，请务必以南京大学教务处、各学院官方网站及辅导员通知为最终依据。

### 如何参与更新

本指南托管在 GitHub，使用 MkDocs Material 构建，任何人都可以通过 Pull Request 来修正错误、补充内容、添加新章节。

- 发现内容已过时？→ 提一个 [Issue](https://github.com/foxsplendid/SurviveNJU-CS/issues) 告诉我们
- 有新内容想贡献？→ 参考 [贡献指南](./contributing.md) 提交 PR
- 不熟悉 GitHub？→ 把内容发给我们，编辑团队来帮你整理

我们相信，一份指南的生命力来自于持续的社区贡献。每一位曾经在南大读书的同学，都是这本书潜在的共同作者。
