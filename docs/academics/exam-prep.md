# 期末备考攻略

泛CS课程可能同时安排闭卷或开卷考试、课程论文、实验验收和项目展示。考试时间、范围与成绩构成随课程和学期变化，应先核对当期教学安排，再制定复习计划。

本篇指南提供通用的复习组织方法，不预测题目，也不替代课程大纲和任课教师通知。

---

## 🛠️ 复习第一步：搜集关键资源

在开始复习前，先确认以下获准使用的材料：

1.  **课程大纲与教师通知**：确认考试日期、地点、形式、范围、允许携带的材料和成绩构成。
2.  **课程组发布的材料**：使用当期课程平台、教师或课程组明确提供的讲义、作业、样题与复习说明。不要使用来源不明的录屏、课件或官方原卷。
3.  **教材与个人笔记**：根据大纲补齐概念和例题。同辈笔记只能辅助理解，需核对版本与错误；未经作者和权利人允许，不要再次分发。

---

## 📐 不同类型课程的针对性备考

根据泛CS专业的课程特点，你可以把复习分为以下三种战术：

### 1. 数理基础课（微积分、线性代数、概率论、离散数学）
*   **备考策略**：**重在做题，切忌只看书。**
*   **方法**：
    *   先把 PPT 上的核心概念和公式过一遍。
    *   使用课程组发布的作业、习题和样题练习；若课程允许使用公开往年题，只把它们当作练习材料，不推断当年命题。
    *   整理错题和概念缺口，回到教材或讲义核对推导条件与解题步骤。

### 2. 系统实践课（ICS PA, 操作系统, 计算机网络）
*   **备考策略**：**理解设计架构，梳理实验细节。**
*   **方法**：
    *   如果当期大纲包含理论考试或实验答辩，按公布范围梳理设计原理、关键数据结构和自己的实现取舍；不要从其他年份经验推断题型。
    *   复习时，重新通读你的实验报告（Report），回忆每个 Lab 你是如何解决 bug 的。
    *   复习讲义中关于系统架构的图表。

### 3. 算法与理论课（数据结构、编译原理）
*   **备考策略**：**掌握核心伪代码，动手推演算法过程。**
*   **方法**：
    *   根据当期大纲和作业要求，练习用伪代码、图表或手算步骤解释算法过程；红黑树、最短路或语法分析等仅是可能的练习主题，不是命题承诺。
    *   在草稿纸上独立推演课程材料中的样例，并检查每一步使用的前提。

## ⏱️ 时间不足时的 3 天应急安排

如果距离考试只剩 3 天，可以用下面的顺序降低遗漏风险。它不能替代平时学习，也不保证通过考试；如果同时有多门考试，应先按考试时间、学分和当前掌握程度排序。

```
Day 1: 建立骨架
快速通读 PPT，遇到看不懂的难点先跳过，重点是明确这门课一共有哪几章，每章的核心概念是什么。

Day 2: 练习与补缺
完成课程组发布的样题、作业或教材练习。根据暴露的问题，回到讲义、教材和作业中的对应内容，优先补齐基础概念。

Day 3: 模拟与纠错
按实际考试形式完成一次限时练习并纠正错题。只整理课程明确要求掌握的公式、定义和易错条件；若考试不允许携带资料，不要把整理页带入考场。
```

!!! tip "考前避坑小贴士"
    *   **按范围取舍**：以当期课程大纲、教师通知和课程组授权材料共同确定范围；PPT 不是对所有课程都适用的“唯一真理源”。
    *   **自行计算所需成绩**：从课程大纲读取各项权重，按“当前已获贡献 + 后续考核贡献”计算目标；不要套用统一的卷面及格阈值，并检查课程是否另设单项及格线。
    *   **保持基本作息**：避免以通宵替代复习，给睡眠、交通和进入考场预留时间；如有健康问题，按医嘱和个人情况安排。
