# 学术诚信

在南京大学泛CS专业的求学过程中，有一个警钟必须时刻长鸣，那就是**学术诚信**。南京大学对作弊、抄袭行为采取“零容忍”态度。一旦触碰学术诚信的红线，你将面临极其严重的惩罚，包括但不限于课程不及格、记过处分、取消保研资格，甚至开除学籍。

在计算机相关课程（尤其是包含大量代码实验的硬核课）中，如何界定“合理参考”与“违规抄袭”，是每个同学必须清晰掌握的界限。

---

## 💻 CS 课程的代码查重机制

南大计算机系和软件学院的多门核心大课（如《计算机系统基础》PA 实验、《操作系统》Lab、《软件工程与计算》大作业）都会使用极其灵敏的**代码防剽窃系统（如 MOSS 算法）**对提交的作业进行全员查重。

*   **MOSS 是如何工作的？**：它不仅能检测一模一样的代码，还能通过抽象语法树（AST）分析检测出**改写变量名、重新格式化、调整函数顺序、改写循环结构（如 for 改为 while）**等伪装手段。即使你花了一小时把别人的代码“重构”了一遍，MOSS 依然能以极高的置信度判定你们的代码存在抄袭。
*   **不仅查本届，还查往届**：你的代码不仅会与同班同学对比，还会与过去 5-10 年所有往届学长学姐提交的代码、以及 GitHub 上能搜到的公开代码库进行交叉对比。

## 🔍 合理参考 vs 直接抄袭的界限

在面对高难度的 Lab（如 OS MiniLab）时，遇到困难向他人学习是正常的，但你必须遵守以下规范：

```
允许的合理参考：
1. 与同学口头讨论算法的思路、解决Bug的整体设计。
2. 阅读网上的教程，理解相关系统调用的作用。
3. 在自己的代码中引用了某段公开的库函数，并在注释中清晰注明了来源（URL）。

禁止的违规抄袭（红线）：
1. 将他人的代码复制粘贴到自己的文件中提交（不论比例大小）。
2. 看着别人的代码，一行一行地“翻译”成自己的代码。
3. 把自己的代码分享给其他同学看、或者直接发给他人参考。
4. 将自己的作业代码在课程结束前公开上传到 GitHub 公共仓库中。
```

!!! danger "“借看代码”也是违规"
    在很多课程组的规则中，**提供代码的一方与抄袭代码的一方同罪**。绝对不要因为同情或友情，将你的实验代码私下发给其他同学。这可能会导致你们两个人的成绩同时归零，并面临教务处处分。

## 🤖 AI 工具（ChatGPT/Gemini等）的使用规范

随着生成式 AI 的普及，如何在完成作业时合理使用 AI 也是我们需要关注的问题：

1.  **辅助解释概念与调试**：你可以让 AI 帮你解释某个复杂的系统调用（例如：“Linux 的 `fork()` 和 `execve()` 有什么区别？”），或者把报错信息发给 AI，让它帮你分析可能的原因。
2.  **严禁直接生成完整作业代码**：让 AI 直接写出大作业的某个完整模块并直接提交，同样存在被查重算法判定为高相似度的风险（因为 AI 针对相同需求的生成模式往往是单一且相似的），且这也违背了你选修这门课程以训练动手能力的初衷。
3.  **遵循课程组细则**：不同的老师对 AI 的使用有不同的尺度要求。开学第一堂课，请仔细确认老师在学术诚信细则（Policy）中对 AI 使用的界定。

---

## 📝 总结：如何保护好自己

*   **尽早开始做实验**：很多抄袭行为都是因为实验拖延到最后一晚，因为焦虑和绝望而铤而走险。如果提前两周开始做，你会有充足的时间向助教和老师提问。
*   **公开仓库注意防护**：如果你将自己写过的优秀项目上传到 GitHub 展示，在课程结束（甚至你毕业）前，**请务必将相关代码仓库设为 Private（私有）**，防止被学弟学妹直接克隆提交，导致你被卷入跨届抄袭的纠纷中。
*   **用注释注明出处**：如果在写代码时参考了 Stack Overflow 上的某段代码片，在代码上方加上注释（如：`// Reference: https://stackoverflow.com/questions/...`）。这能证明你是合理引用，而非恶意抄袭。
